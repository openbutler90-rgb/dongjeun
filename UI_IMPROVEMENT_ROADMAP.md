## 🚨 1순위: 관리자 페이지 (비효율의 끝판왕)

### 1-1. AdminJoinRequests.tsx - 가입신청 관리

| # | 현재 요소 | 문제 | 개선 방향 |
|---|----------|------|----------|
| 1 | **승인/거절/삭제 버튼 3개** (`flex-1 py-2 bg-green-500...`) | 카드 하단에 큰 감정색 버튼 3개 나열, 시각적 혼란 | **⋮ 액션** 버튼 하나로 → 드롭다운: 승인 / 거절 / 삭제 |
| 2 | **필터 탭 4개 + 전체승인 버튼** | 한 줄에 버튼 5개 | **⋮ 필터/액션** 메뉴 하나로 → "전체/대기/승인/거절" + 구분선 + "대기자 전체승인" |
| 3 | **자동승인 ON/OFF 영역** (`p-4 rounded-2xl border-2`) | 너무 큰 토글 카드, 시각적 무게 과다 | 작은 **토글 스위치**만. 설명은 `?` 툴팁 아이콘 |
| 4 | **가입신청 카드** (`rounded-2xl p-4`) | 카드 하나가 너무 큼, 정보 밀도 낮음 | `rounded-xl p-3` 축소. 접힌 상태에서는 답변 숨김 |
| 5 | **프로필 사진** (`w-9 h-9 rounded-full`) | 36px로 크고 border 등 무게감 있음 | `w-7 h-7` 축소 또는 이니셜 텍스트만 |
| 6 | **"펼치기 ▼ / 접기 ▲"** | 텍스트로 공간 차지 | **작은 ▼ 아이콘**만 (12px) |
| 7 | **승인 모달** (`max-w-2xl text-base font-black`) | 너무 큰 모달, 관리자 눈 피로 | `max-w-md`, `text-sm font-bold` |

### 1-2. AdminPage.tsx - AI 발행/청소

| # | 현재 요소 | 문제 | 개선 방향 |
|---|----------|------|----------|
| 1 | **AI 발행 버튼 2개** (`🗑️ 청소` / `🚀 발행`) | 긴 텍스트 버튼 2개가 큰 영역 | **⋮ 작업** 메뉴 하나 → "AI 발행 / 청소 미리보기 / 일괄 삭제" |
| 2 | **카테고리 선택 그리드** (`grid-cols-2 sm:grid-cols-4`) | 10개 카테고리가 박스로 깔림 | **접힌 상태** ("5개 선택됨 ▼") → 클릭 시 펼침. 또는 드롭다운 |
| 3 | **청소 모달 카테고리 버튼** | 모달 상단에 카테고리별 버튼 일렬 | **⋮ 필터** 버튼 → 드롭다운에서 일괄 보존/해제 |
| 4 | **게시물 체크박스 버튼** (`w-6 h-6 rounded-md border-2`) | 버튼으로 만든 체크박스, 클릭 영역 큼 | **작은 `<input type="checkbox">`** (w-4 h-4) |
| 5 | **삭제/취소 버튼** (`px-6 py-2.5 bg-rose-500`) | 하단에 큰 감정적 버튼 | **텍스트 링크** 스타일 ("취소" / "삭제 확정") |
| 6 | **청소 모달 전체** (`max-w-2xl max-h-[85vh]`) | 모바일에서 화면 다 먹음 | `max-w-lg`, 모바일은 `h-screen` |

### 1-3. AdminPage.tsx - 배너 관리

| # | 현재 요소 | 문제 | 개선 방향 |
|---|----------|------|----------|
| 1 | **배너 필드 50+개** | focalX/Y, titleX/Y, logoX/Y... 폭발 | **탭 분리**: 기본정보 / 위치·크기 / 고급설정 |
| 2 | **모든 필드 한 페이지** | 스크롤이 엄청남 | **아코디언** 또는 **탭**으로 분리 |
